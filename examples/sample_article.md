# wxAiPost 应该怎么借鉴，而不是照搬

> [!TIP] 核心判断
> wxAiPost 更适合当发布链路和后台骨架参考，不适合当最终排版核心。

一个真正稳定的公众号生产系统，不应该把“写作、排版、发布、记录”揉成一个不可拆的脚本。

## 1. 账号和 token 要数据库化

公众号多账号运营时，最容易出问题的是 token 串号、过期、重复刷新。正确做法是把账号表、token 表和发布记录表分开。

## 2. 排版要先本地预览，再进入草稿

排版层应该输出微信兼容的 inline HTML。这里不依赖 `<style>`、class 或外链 CSS，因为这些在微信编辑器里很容易丢失。

## 3. 发布链路不要一步到群发

更稳的顺序是：

1. 渲染本地预览
2. 上传正文图片
3. 上传封面素材
4. 创建草稿
5. 发送手机预览
6. 再提交发布
7. 轮询状态并记录 article_id / url

## 结论

把 wxAiPost 当后台雏形，把 WeMD 当排版审美参考，把 Python SDK 当接口层参考，最后组合成自己的 Python-only 系统。
